# Шаблоны WB-rules

Папка содержит шаблоны сценариев Wiren Board для повторного использования между объектами.

Шаблоны используются как заготовки логики. Перед применением на объекте каждый шаблон адаптируется под подтверждённые каналы, оборудование, уставки и фактические подключения конкретного объекта.

## Разделы

```text
Templates/WB-rules/
├─ heating/
├─ ventilation/
├─ lighting/
├─ water/
├─ gates/
└─ alarms/
```

## Назначение разделов
- `heating/` — отопление, термостаты, подмесы, насосы, подпитка.
- `ventilation/` — вентиляция, вытяжки, приточные установки.
- `lighting/` — освещение, сцены, диммирование, мастер-выключатели.
- `water/` — водоснабжение, протечки, давление, подпитка.
- `gates/` — ворота, калитки, приводы.
- `alarms/` — аварии, уведомления, диагностика.

## Правило шаблонов

Шаблон WB-rules не должен содержать объектные факты:
```text
MQTT-каналы конкретного объекта;
Modbus-адреса;
slave_id;
device id;
номера входов и выходов;
уставки конкретного объекта;
названия помещений конкретного объекта;
фактические подключения конкретного объекта.
```
Если в шаблоне нужны каналы, использовать условные имена или отдельный блок конфигурации, который заменяется при адаптации под объект.

## Применение шаблона

Перед применением шаблона нужно:

1. Прочитать файлы текущего объекта.
2. Подтвердить реальные MQTT-каналы.
3. Проверить, кто пишет в исполнительные каналы.
4. Убедиться, что не нарушается правило:
один физический канал = один писатель
5. Адаптировать конфигурационный блок шаблона под объект.
6. Проверить совместимость с wb-rules.

## Связь со стандартами

При подготовке шаблонов использовать:
```text
EIM/Standards/WB_coding_standard.md
EIM/Standards/WB_logging_standard.md
## Логика и интерфейс
```
Для инженерных систем логика должна оставаться в Wiren Board:
```text
Wiren Board = логика, защита, исполнительные команды, журнал
```
Интерфейсные системы, например Sprut.hub, должны использоваться для отображения и управления:
```text
Sprut.hub = отображение, уставки, режимы, пользовательское управление
```
Шаблон WB-rules должен быть самостоятельной заготовкой логики и не должен зависеть от неподтверждённых данных другого объекта.
