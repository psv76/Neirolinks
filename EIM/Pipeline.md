# Pipeline разработки системы автоматизации

Цель процесса — связать физический проект (DWG) и программную систему (скрипты Wiren Board) и минимизировать ручную работу при наладке объектов.

Разработка делится на несколько этапов.

## 1. Проектирование

Проект выполняется в AutoCAD с использованием стандартных блоков.

Каждое устройство имеет набор атрибутов:

- ID
- DEVICE_TYPE
- LOAD_TYPE
- CONTACT
- BOARD
- ROOM
и т.д.

Ключевым элементом является канонический идентификатор канала.

Формат:

MODULE.CHANNEL

пример

- A06.K1
- A06.in1
- A05.K2

где

- A06 — модуль в щите
- K1 — канал реле
- in1 — вход

Этот идентификатор используется далее во всех слоях системы.

## 2. Парсинг проекта

Python-утилита анализирует DWG.

Из блоков извлекаются:

- идентификатор линии
- тип устройства
- контакт
- нагрузка
- помещение
- щит

На основе этих данных формируется **Кабельный журнал**

Пример

| ID | устройство | контакт |
|---|---|---|
|208|Выключатель гостиная|A06.in1|
|324|Люстра гостиная|A06.K1|

На этом этапе система всё ещё описывает только физическую инфраструктуру.

## 3. Определение автоматизации

Инженер определяет:
- какие узлы и системы нужно автоматизировать
- какие каналы участвуют в автоматизации

Это фактически означает: какие скрипты будут использоваться

Например:

Узел: 

Включение света по датчику движения

Каналы:
- A06.K1 — освещение
- A06.in1 — выключатель
- A01.in3 — датчик движения


## 4. Формирование паспорта объекта

Python-утилита формирует паспорт объекта.

Паспорт — это техническое задание для ChatGPT, содержащее минимальную информацию, необходимую для генерации программной части системы.

Пример структуры:

```
object: 05_16_Иcеть

modules:

  A06:
    model: WB-MR6C
    modbus: 85

channels:

  A06.K1:
    type: relay
    name: Люстра гостиная

  A06.in1:
    type: input
    name: Выключатель гостиная

scripts_required:

  - lighting
  - night_light
```

Паспорт не содержит:

- кабельный журнал
- схемы
- код

Он описывает контекст объекта для генерации логики системы.

## 5. Генерация скриптов

Паспорт объекта публикуется в GitHub.

В репозитории также хранится библиотека шаблонов скриптов.

пример

```
scripts_templates
 ├ lighting
 ├ heating
 ├ ventilation
 ├ alarms
```

ChatGPT получает:
```
паспорт объекта
+
библиотеку шаблонов
```
и генерирует
```
скрипты для конкретного объекта
```
## 6. Настройка контроллера

На этапе наладки выполняется:

- назначение Modbus-адресов
- настройка wb-mqtt-serial.conf
- привязка MQTT id устройств к проектным модулям

пример
```
slave_id = 85
id = A06
name = "A06 WB-MR6C (85)"
```
После этого MQTT-каналы совпадают с проектными идентификаторами:
```
A06.K1 → /devices/A06/controls/K1
A06.in1 → /devices/A06/controls/Input 1
```
Итоговая архитектура

Получается единая цепочка:
```
DWG
↓
Python parser
↓
модель каналов
↓
паспорт объекта
↓
ChatGPT
↓
скрипты
↓
Wiren Board runtime
```
## Ключевой принцип системы

Используется единый идентификатор канала:
```
MODULE.CHANNEL
```
пример
```
A06.K1
A06.in1
```
Он одинаково используется:

- в проекте
- в MQTT
- в скриптах
- в документации

Это позволяет связать:
```
физическую систему
runtime систему
программную систему
```
